# Google Drive 동기화 설정 가이드 (개발자용)

## 개요
누니메모의 Google Drive 동기화 기능을 위한 개발자 설정 가이드입니다. 개발자가 한 번만 설정하면 모든 사용자가 Google 로그인만으로 동기화를 사용할 수 있습니다.

## 1. Google Cloud Console 설정

### 1.1 프로젝트 생성
1. [Google Cloud Console](https://console.cloud.google.com/) 접속
2. 상단 프로젝트 선택 → **새 프로젝트**
3. 프로젝트 이름: `NuniMemo`
4. **만들기** 클릭

### 1.2 Google Drive API 활성화
1. 좌측 메뉴 → **API 및 서비스** → **라이브러리**
2. "Google Drive API" 검색
3. **Google Drive API** 선택 → **사용** 클릭

### 1.3 OAuth 동의 화면 구성
1. 좌측 메뉴 → **API 및 서비스** → **OAuth 동의 화면**
2. **User Type**: **외부** 선택
3. **만들기**
4. **앱 정보**:
   - **앱 이름**: `NuniMemo`
   - **사용자 지원 이메일**: 본인 이메일
   - **개발자 연락처 정보**: 본인 이메일
5. **범위**: 지금은 건너뛰기
6. **테스트 사용자**: 본인 Google 계정 추가
7. **저장 후 계속**

### 1.4 앱 게시
1. **OAuth 동의 화면** → **앱 게시**
2. **앱 확인용** 선택
3. **게시** 클릭

### 1.5 클라이언트 ID 생성
1. 좌측 메뉴 → **API 및 서비스** → **사용자 인증 정보**
2. **+ 사용자 인증 정보 만들기** → **OAuth 2.0 클라이언트 ID**
3. **애플리케이션 유형**: **데스크톱 앱**
4. **이름**: `NuniMemo Desktop Client`
5. **만들기**

### 1.6 인증 파일 다운로드
1. 생성된 클라이언트 ID 옆 **다운로드** 버튼 클릭
2. 파일 이름이 `client_secret_...json`으로 다운로드됨
3. 파일 이름을 **`credentials.json`**으로 변경

## 2. 배포용 인증 파일 설정

### 2.1 파일 위치
```
e:\Pjt\NuniMemo\mycloudmemo\sync\credentials.json
```

### 2.2 파일 배치
1. 다운로드한 `credentials.json`을 위 경로에 복사
2. 이 파일은 앱과 함께 배포됨

### 2.3 보안 주의사항
- `credentials.json`은 공개적으로 노출되면 안 됨
- GitHub 등에 올리지 말 것
- 배포 시에는 적절한 보안 조치 필요

## 3. 사용자 인증 흐름

### 3.1 최초 실행
1. 사용자가 **🔄 Google Drive 동기화** 버튼 클릭
2. 브라우저에서 Google 로그인 화면 자동 열림
3. 사용자가 Google 계정으로 로그인
4. 권한 동의 (Google Drive 접근)
5. 인증 성공 후 자동으로 동기화 시작

### 3.2 이후 실행
- 인증 정보가 저장되어 있으므로 바로 동기화 시작
- 토큰 만료 시 자동 갱신

## 4. 테스트 확인

### 4.1 개발자 테스트
```powershell
# 앱 실행
& e:/Pjt\NuniMemo\venv/Scripts/python.exe e:/Pjt\NuniMemo/main.py

# 동기화 버튼 클릭
# 브라우저에서 Google 로그인 확인
# 동기화 성공 메시지 확인
```

### 4.2 사용자 테스트
- 다른 PC에서 앱 실행
- 동기화 기능이 정상 작동하는지 확인
- Google 로그인만으로 동기화되는지 확인

## 5. 배포 준비

### 5.1 필수 파일
- `credentials.json` (개발자 설정용)
- 앱 실행 파일
- 관련 라이브러리

### 5.2 사용자 안내
- Google Drive 동기화 사용법
- 최초 인증 절차 안내
- 개인정보 보호 정책

## 6. 문제 해결

### 6.1 일반 오류
- **인증 실패**: `credentials.json` 확인
- **권한 오류**: OAuth 동의 화면 확인
- **네트워크 오류**: 인터넷 연결 확인

### 6.2 디버깅
- 콘솔 로그 확인
- Google Cloud Console 오류 메시지 확인
- 사용자 인증 토큰 상태 확인
